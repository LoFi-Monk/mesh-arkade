<project_specification>
  <project_name>Mesh ARKade</project_name>

  <overview>
    Mesh ARKade is a decentralized game preservation platform built entirely on the Pear Runtime (Holepunch ecosystem). It operates as a CLI-first, P2P-native &quot;museum-quality&quot; archive for retro game ROMs. The application provides a headless terminal interface for managing game library mounts, discovering supported retro systems via Libretro&apos;s DAT database, seeding game metadata (wishlist) from No-Intro/Redump DAT files into a local Hyperbee database, searching the seeded catalog, and fetching ROMs via a triple-layer P2P discovery stack (Hyperswarm → IPFS → BitTorrent DHT). The architecture follows a singleton CoreHub pattern with JSON-RPC style request dispatch, a dual-runtime abstraction layer (Bare runtime vs Node.js), and a modular CLI with individually testable command handlers. A React/Vite GUI mode exists as a secondary interface (Pear Electron) but the primary focus is the headless CLI where both human users and AI agents are equal citizens. The project uses an OpenSpec workflow for change management and enforces SOLID, DRY, and TDD principles.
  </overview>

Technical Implementation:

**Root Cause Analysis:**
`bare-type` uses a native addon loaded via `require.addon` — a Bare runtime API that does not exist in the Node.js/Vitest test environment.
